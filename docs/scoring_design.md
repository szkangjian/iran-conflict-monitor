# 冲突严重程度定量评分设计文档

## 背景与目标

本系统从多个新闻源持续抓取涉及伊朗冲突的事件，目标是将碎片化的新闻信号聚合为一个**可追踪、可比较的量化指标**——冲突严重程度综合评分（0–100分），用于：

- 判断当前局势所处的烈度区间
- 识别升级或降温的早期信号
- 为后续时间序列分析和预测提供基础数据

100分代表理论最严重状态（全面核战争边缘），0分代表完全平静。

---

## 核心设计原则：快照重算，无状态记忆

**每次评分都基于当前时间窗口内的事件全量重新计算，不依赖历史评分状态。**

这一选择基于以下考虑：

| 方案 | 描述 | 放弃原因 |
|---|---|---|
| 加减分累积制 | 升级加分，缓和减分，历史状态滚动 | 路径依赖：早期分数会无限累积，沉默期（无新闻）无法自动修正 |
| 快照重算制（采用） | 每次用时间窗口内所有事件重新计算 | 可解释性强，缓和信号天然降低窗口内平均分 |
| 双轨基线制 | 慢变基线 + 快变信号叠加 | 基线估计需要大量结构化数据，当前阶段过度依赖假设 |

**缓和信号的处理方式**：停火、谈判、撤军等事件作为**负权重事件**进入计算窗口，而非在总分上做减法。这样，缓和信号越多，窗口内加权平均自然越低，逻辑干净，无需引入"当前分数"的概念。

---

## 评分架构

总分由**五个维度**加权合成，每个维度独立打分（0–10），再按权重汇总后映射到0–100。

```
总分 = Σ(维度得分 × 权重) × 10
```

| 维度 | 权重 | 核心问题 |
|---|---|---|
| D1 军事行动强度 | 30% | 当前窗口内发生了什么，强度如何？ |
| D2 升级趋势 | 25% | 本周与上周相比，局势在变好还是变坏？ |
| D3 核与战略武器风险 | 20% | 距不可逆门槛有多近？ |
| D4 地理与参与方扩散 | 15% | 冲突是否在蔓延到更多国家和行为体？ |
| D5 外交状态 | 10% | 有无降温机制在运作？ |

### 强制规则

> 当 D3（核与战略武器风险）得分 ≥ 7 时，总分强制不低于 70。
>
> 理由：核门槛一旦逼近，其他维度的缓和信号已无实质意义。

## 各维度详细设计

### D1 — 军事行动强度（权重 30%）

**计算窗口**：过去 48 小时内的事件

#### 步骤一：为每个事件分配基础分

升级性事件为正分，缓和性事件为负分，两者在同一窗口内共同参与计算。

**升级事件（正分）**

| 事件类型 | 判断依据（关键词/标签） | 基础分 |
|---|---|---|
| 核打击 / 正式宣战 | nuclear + attack / war declared | +10 |
| 弹道导弹实战使用 | ballistic missile + strike/used | +8 |
| 空袭重要基础设施 | airstrike + bridge/power/refinery | +7 |
| 无人机攻击 | drone + attack | +6 |
| 定点暗杀 | assassination / targeted killing | +6 |
| 代理武装大规模交火 | proxy + hezbollah/hamas/houthi + attack | +5 |
| 制裁升级 / 海上封锁 | sanctions escalation / blockade | +4 |
| 军事演习 / 兵力部署 | military exercise / troops deployed | +3 |
| 外交警告 / 威胁声明 | warning / threat / ultimatum | +2 |
| 情报活动 / 间谍事件 | spy / intelligence operation | +1 |

**缓和事件（负分）**

| 事件类型 | 判断依据（关键词） | 基础分 |
|---|---|---|
| 停火协议正式签署 | ceasefire agreement / signed | -6 |
| 正式谈判启动 | negotiations / talks begin/start | -4 |
| 军队撤退 / 降级行动 | troops withdrawal / de-escalation | -3 |
| 外交渠道恢复 | diplomatic contact / envoy restored | -2 |
| 单方面停火声明 | unilateral ceasefire / pause | -2 |
| 人道主义走廊开放 | humanitarian corridor / aid access | -1 |

#### 步骤二：计算维度得分

```
net_sum = Σ(窗口内所有事件的基础分)   # 正负相抵
peak    = 窗口内所有正分事件的最高分
volume  = log2(正分事件数 + 1)，上限 3

D1 = min(10, max(0,  peak × 0.5 + net_sum × 0.3 + volume × 0.5))
```

设计意图：
- `peak` 捕捉单一极端事件的冲击
- `net_sum` 体现缓和信号的抵消作用
- `volume` 反映事件密集度，但设上限防止刷屏
- 整体下限为 0，不允许 D1 为负（缓和不能倒扣军事烈度）

### D2 — 升级趋势（权重 25%）

**计算窗口**：对比"本周（近7天）"与"上周（7–14天前）"

**加权事件量**：将缓和事件的负分也纳入计算，反映真实的净信号强度。

```
本周日均净分 = Σ(本周事件基础分) / 7
上周日均净分 = Σ(上周事件基础分) / 7

Δ = (本周日均净分 - 上周日均净分) / (|上周日均净分| + 0.1)

D2 得分映射：
  Δ > 1.0   →  10  （翻倍以上，急剧升级）
  Δ > 0.5   →  8
  Δ > 0.2   →  6
  Δ > 0.0   →  5   （轻微升级）
  Δ ≈ 0     →  4   （平稳）
  Δ < -0.2  →  3
  Δ < -0.5  →  2   （明显降温）
  Δ < -0.8  →  0   （大幅降温）
```

说明：分母取绝对值，防止上周净分为负时方向颠倒。

### D3 — 核与战略武器风险（权重 20%）

此维度采用**累加制**，各子项独立判断后叠加，上限 10。

**升级子项（正分）**

| 子项 | 判断依据 | 得分 |
|---|---|---|
| 铀浓缩达武器级（>90%） | enrichment + 90% | +5 |
| 铀浓缩高浓缩（60–90%） | enrichment + 60%/80% | +3 |
| 中等浓缩（20–60%） | enrichment + 20%/40% | +1 |
| 弹道导弹 / 高超音速测试或使用 | ballistic / hypersonic | +3 |
| 核设施遭到攻击 | natanz / fordo + attack/strike | +4 |
| 官方明确核威胁 | nuclear + threat / weapon | +2 |
| IAEA 访问受阻 | IAEA + blocked/refused | +1 |

**缓和子项（负分）**

| 子项 | 判断依据 | 得分 |
|---|---|---|
| IAEA 重新获准进入核设施 | IAEA + access / inspectors | -2 |
| 铀浓缩活动暂停声明 | enrichment + suspended/halted | -3 |
| 核协议框架重启 | JCPOA / nuclear deal + revived/resumed | -2 |

```
D3 = min(10, max(0, Σ子项得分))
```

**强制规则触发条件**：D3 ≥ 7 → 总分强制 ≥ 70

---

### D4 — 地理与参与方扩散（权重 15%）

**子项 A：卷入国家/地区数**

| 状态 | 得分 |
|---|---|
| 仅伊朗 + 以色列 + 美国三方 | 3 |
| + 海湾国家（沙特、UAE、科威特等）受波及 | 5 |
| + 巴基斯坦 / 土耳其 / 伊拉克卷入 | 7 |
| + 欧洲国家直接介入 或 中俄明确表态支持一方 | 9 |

**子项 B：代理武装活跃度**

| 状态 | 得分 |
|---|---|
| 1 个代理武装活跃 | +1 |
| 2 个同时活跃 | +2 |
| 3 个（Hezbollah + Hamas + Houthi）均活跃 | +3 |

**子项 C：冲突收缩信号（负分）**

| 状态 | 得分 |
|---|---|
| 代理武装宣布停火或退出 | -1 / 个 |
| 第三国宣布中立或调停 | -1 |

```
D4 = min(10, max(0, 子项A + 子项B + 子项C))
```

---

### D5 — 外交状态（权重 10%）

反向指标：此维度完全基于当前窗口（近 7 天）的外交类事件快照，无记忆。

**正式谈判 / 协议信号（降分）**

| 信号 | 关键词 | 得分 |
|---|---|---|
| 正式谈判进行中，有具体议程 | talks / negotiations + underway | 1 |
| 第三方积极斡旋 | mediation / broker + active | 2 |
| 非正式外交接触 | envoy / diplomat + met/visited | 3 |
| 无明显外交活动（中性默认） | — | 5 |

**外交恶化信号（升分）**

| 信号 | 关键词 | 得分 |
|---|---|---|
| 谈判破裂声明 | talks + collapsed/failed | 6 |
| 外交渠道基本断绝 | diplomatic + severed/cut | 7 |
| 相互驱逐外交官 | expelled / persona non grata | 8 |
| 外交关系完全破裂 + 战争状态 | war + declared / relations + broke | 10 |

取窗口内**最强信号**（正式谈判与外交恶化不可叠加，取最近发生的主导信号）。默认值 5。

## 总分合成

```python
raw_score = (
    D1 * 0.30 +
    D2 * 0.25 +
    D3 * 0.20 +
    D4 * 0.15 +
    D5 * 0.10
) * 10

# 强制规则：核风险逼近时总分不低于 70
if D3 >= 7:
    final_score = max(raw_score, 70)
else:
    final_score = raw_score

final_score = round(min(100, max(0, final_score)), 1)
```

---

## 分数区间与含义

| 区间 | 标签 | 典型状态描述 |
|---|---|---|
| 0–20 | 低度紧张 | 外交博弈为主，偶发言辞冲突，无军事行动 |
| 20–40 | 代理对抗 | 制裁施压 + 代理武装零星交火，直接对抗风险低 |
| 40–60 | 直接军事对抗 | 导弹 / 空袭已发生，局势不稳但仍有控制空间 |
| 60–80 | 全面战争 | 双方大规模互击，关键基础设施受损，第三方卷入 |
| 80–100 | 核门槛边缘 | 战略武器威胁明确，局势接近不可逆转 |

## 数据存储设计

### `events` 表新增字段

```sql
ALTER TABLE events ADD COLUMN event_score REAL DEFAULT NULL;
-- 存储单事件基础分（正分或负分），供各维度聚合使用
```

### 新表：`daily_scores`

```sql
CREATE TABLE daily_scores (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    date        TEXT    UNIQUE NOT NULL,   -- YYYY-MM-DD（UTC）
    score       REAL    NOT NULL,          -- 综合总分 0–100
    d1          REAL,                      -- 各维度分项，便于调试和回溯
    d2          REAL,
    d3          REAL,
    d4          REAL,
    d5          REAL,
    nuclear_override INTEGER DEFAULT 0,    -- 是否触发了核强制规则
    computed_at TEXT    NOT NULL
);
```

每小时重算当天记录（upsert），保留历史序列供趋势图使用。

## API 扩展

### `GET /api/score`

返回当前综合评分及各维度明细：

```json
{
  "score": 63.5,
  "level": "全面战争",
  "dimensions": {
    "d1": 7.2,
    "d2": 6.5,
    "d3": 5.0,
    "d4": 6.0,
    "d5": 7.0
  },
  "nuclear_override": false,
  "computed_at": "2026-04-03T15:00:00Z"
}
```

### `GET /api/score/history?days=30`

返回近 N 天的历史评分序列，供折线图展示：

```json
[
  {"date": "2026-04-03", "score": 63.5, "d1": 7.2, "d2": 6.5, ...},
  {"date": "2026-04-02", "score": 58.1, ...}
]
```

## 局限性与说明

1. **数据来源偏差**：评分基于英文媒体报道，存在报道延迟和选择性偏差，极端事件发生后数小时内评分可能偏低。

2. **缓和信号的对称性问题**：负分事件的权重目前基于判断设定，建议在积累足够历史数据后通过回测校准正负分比例。

3. **D5 外交状态**：窗口内同时存在谈判和外交恶化信号时，取"最近发生的主导信号"的规则较为粗糙，未来可改为加权平均。

4. **核浓缩度**：铀浓缩百分比需从新闻正文中提取，可能存在误读，未来可接入 IAEA 官方数据源。

5. **评分频率**：建议每小时重新计算一次，而非实时，以平滑新闻噪声。

6. **基准校准**：0分和100分为理论极值，建议以历史事件校准参考：
   - 2020年1月苏莱曼尼事件（暗杀+弹道导弹反击）≈ 65–70分
   - 2015年JCPOA签署期间 ≈ 15–20分
   - 2024年4月伊朗直接导弹袭击以色列 ≈ 75–80分
