# 冲突严重程度定量评分设计文档

## 背景与目标

本系统从多个新闻源持续抓取涉及伊朗冲突的事件，目标是将碎片化的新闻信号聚合为一个**可追踪、可比较的量化指标**——冲突严重程度综合评分（0–100分），用于：

- 判断当前局势所处的烈度区间
- 识别升级或降温的早期信号
- 为后续时间序列分析和预测提供基础数据

100分代表理论最严重状态（全面核战争边缘），0分代表完全平静。

---

## 评分架构

总分由**五个维度**加权合成，每个维度独立打分（0–10），再按权重汇总后映射到0–100。

```
总分 = Σ(维度得分 × 权重) × 10
```

| 维度 | 权重 | 核心问题 |
|---|---|---|
| D1 军事行动强度 | 30% | 正在发生什么行动？ |
| D2 升级趋势 | 25% | 局势在变好还是变坏？ |
| D3 核与战略武器风险 | 20% | 距不可逆门槛有多近？ |
| D4 地理与参与方扩散 | 15% | 冲突是否在蔓延？ |
| D5 外交状态 | 10% | 有无降温机制在运作？ |

### 强制规则

> 当 D3（核与战略武器风险）得分 ≥ 7 时，总分强制不低于 70。
>
> 理由：核门槛一旦逼近，其他维度的缓和信号已无实质意义。

---

## 各维度详细设计

### D1 — 军事行动强度（权重 30%）

**计算窗口**：过去 48 小时内的事件

**步骤一：为每个事件分配基础分**

| 事件类型 | 判断依据（关键词/标签） | 基础分 |
|---|---|---|
| 核打击 / 正式宣战 | nuclear + attack / war declared | 10 |
| 弹道导弹实战使用 | ballistic missile + strike | 8 |
| 空袭重要基础设施 | airstrike + bridge/power/dam | 7 |
| 无人机攻击 | drone + attack | 6 |
| 定点暗杀 | assassination | 6 |
| 代理武装大规模交火 | proxy + hezbollah/hamas/houthi + attack | 5 |
| 制裁 / 海上封锁 | sanctions / blockade | 4 |
| 军事演习 / 部署 | military exercise / troops deployed | 3 |
| 外交警告 / 威胁声明 | warning / threat | 2 |
| 情报活动 / 间谍 | spy / intelligence | 1 |

**步骤二：计算维度得分**

```
peak   = 窗口内所有事件的最高基础分
volume = log2(窗口内事件数 + 1)，上限 3（即事件数 ≥ 7 时饱和）

D1 = min(10,  peak × 0.7 + volume × 1.0)
```

设计意图：单一极端事件（peak）权重更高；大量低级别事件也能推高分数，但有饱和限制，避免新闻刷屏导致虚高。

---

### D2 — 升级趋势（权重 25%）

**计算窗口**：对比"本周（近7天）"与"上周（7–14天前）"的加权事件量

**加权事件量** = Σ(事件基础分) / 窗口天数（日均）

```
Δ = (本周日均加权量 - 上周日均加权量) / (上周日均加权量 + 0.1)

D2 得分映射：
  Δ > 1.0   →  10  （翻倍以上，急剧升级）
  Δ > 0.5   →  8
  Δ > 0.2   →  6
  Δ > 0.0   →  5  （轻微升级）
  Δ = 0     →  4  （平稳）
  Δ < -0.2  →  3
  Δ < -0.5  →  2  （明显降温）
  Δ < -0.8  →  0  （大幅降温）
```

设计意图：趋势比绝对量更能反映局势走向。分母加 0.1 避免上周数据为零时除法溢出。

---

### D3 — 核与战略武器风险（权重 20%）

此维度采用**累加制**，各子项独立判断后叠加，上限 10。

| 子项 | 判断依据 | 得分 |
|---|---|---|
| 铀浓缩度达武器级（>90%） | enrichment + 90% | +5 |
| 铀浓缩度高浓缩（60–90%） | enrichment + 60%/80% | +3 |
| 中等浓缩（20–60%） | enrichment + 20%/40%/60% | +1 |
| 弹道导弹 / 高超音速测试或使用 | ballistic / hypersonic | +3 |
| 核设施遭到攻击 | natanz / fordo + attack/strike | +4 |
| 官方发出明确核威胁 | nuclear + threat / weapon | +2 |
| IAEA 访问受阻 | IAEA + blocked/refused | +1 |

**强制规则触发条件**：D3 ≥ 7 → 总分强制 ≥ 70

---

### D4 — 地理与参与方扩散（权重 15%）

**子项 A：卷入国家/地区数**

| 状态 | 得分 |
|---|---|
| 仅伊朗 + 以色列 + 美国三方 | 3 |
| + 海湾国家（沙特、UAE、科威特等）受波及 | 5 |
| + 巴基斯坦 / 土耳其 / 伊拉克卷入 | 7 |
| + 欧洲国家直接介入 或 中俄明确表态支持一方 | 9 |

**子项 B：代理武装活跃度**

| 状态 | 加分 |
|---|---|
| 1 个代理武装活跃 | +1 |
| 2 个同时活跃 | +2 |
| 3 个（Hezbollah + Hamas + Houthi）均活跃 | +3 |

```
D4 = min(10, 子项A + 子项B)
```

---

### D5 — 外交状态（权重 10%）

此维度为**反向指标**：外交越活跃，得分越低，对总分贡献越小。

| 状态 | 得分 |
|---|---|
| 正式谈判进行中，有具体议程 | 0–2 |
| 非正式接触 / 第三方斡旋 | 3–4 |
| 谈判暂停，渠道仍存在 | 5 |
| 外交渠道基本断绝 | 6–7 |
| 相互驱逐外交官 | 8 |
| 外交关系完全破裂 + 战争状态 | 10 |

**说明**：D5 主要依靠人工标注或关键词（negotiations / talks / ceasefire 为低分信号；expelled / severed / war 为高分信号），初期可设默认值 5（中性），随新闻动态调整。

## 总分合成

```python
raw_score = (
    D1 * 0.30 +
    D2 * 0.25 +
    D3 * 0.20 +
    D4 * 0.15 +
    D5 * 0.10
) * 10

# 强制规则
if D3 >= 7:
    final_score = max(raw_score, 70)
else:
    final_score = raw_score

final_score = round(min(100, max(0, final_score)), 1)
```

---

## 分数区间与含义

| 区间 | 标签 | 典型状态描述 |
|---|---|---|
| 0–20 | 低度紧张 | 外交博弈为主，偶发言辞冲突，无军事行动 |
| 20–40 | 代理对抗 | 制裁施压 + 代理武装零星交火，直接对抗风险低 |
| 40–60 | 直接军事对抗 | 导弹 / 空袭已发生，局势不稳但仍有控制空间 |
| 60–80 | 全面战争 | 双方大规模互击，关键基础设施受损，第三方卷入 |
| 80–100 | 核门槛边缘 | 战略武器威胁明确，局势接近不可逆转 |

## 数据存储设计

在现有 `events` 表基础上新增：

### 新表：`daily_scores`

```sql
CREATE TABLE daily_scores (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    date        TEXT    UNIQUE NOT NULL,   -- YYYY-MM-DD
    score       REAL    NOT NULL,
    d1          REAL,   -- 各维度分项，便于调试和回溯
    d2          REAL,
    d3          REAL,
    d4          REAL,
    d5          REAL,
    computed_at TEXT    NOT NULL
);
```

每次评分后写入当天记录（upsert），保留历史序列供趋势图使用。

### `events` 表新增字段

```sql
ALTER TABLE events ADD COLUMN event_score REAL DEFAULT NULL;
```

存储单事件基础分，供 D1/D2 聚合计算使用。

## 局限性与说明

1. **数据来源偏差**：评分基于英文媒体报道，存在报道延迟和选择性偏差，极端事件发生后数小时内评分可能偏低。

2. **D5 外交状态**：目前依赖关键词推断，准确度有限，建议定期人工校准。

3. **核浓缩度**：铀浓缩百分比需从新闻正文中提取，可能存在误读，未来可接入 IAEA 官方数据。

4. **评分频率**：建议每小时重新计算一次，而非实时，以平滑新闻噪声。

5. **基准校准**：0分和100分为理论极值，实际历史峰值（如2020年苏莱曼尼事件）可作为校准参考点。
